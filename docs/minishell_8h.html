Prints an error message to stderr when a command is not found and exits with the last recorded exit status of the shell.

  • Uses the write() system call for direct output.

Parameters

  • cmd->minishell->exit_status stores the exit status.

Determines the executable path for a given command.

This function checks whether the command is:

  • An absolute or relative path (/bin/ls, ./script.sh).
  • Located in the system's PATH. If found, the command's binary path is stored in cmd->binary.

Returns

◆ init_mshell()

t_mshell* init_mshell ( char **  envp)

Initializes the Minishell structure.

This function initializes the shell by:

  • Allocating memory for the t_mshell structure.
  • Setting up the environment variables.
  • Initializing the hash table for variable storage.
  • Loading built-in commands.

If any initialization step fails, the allocated memory is freed and NULL is returned.

Parameters
envpThe array of environment variables from the system.
Returns
A pointer to the initialized t_mshell structure, or NULL on failure.

Retrieves the value of an environment variable from the hash table.

This function searches for an environment variable stored in the Minishell hash table by computing its hash index and traversing the linked list of stored variables.

Parameters
mshellPointer to the Minishell structure containing the hash table.
keyThe name of the environment variable to retrieve.
Returns
Pointer to the value of the variable if found, or NULL if not found.
Note
All values stors in the hash table for faster search, export, and unset

Initializes the list of built-in shell commands.

This function allocates memory for an array of built-in command names and populates it with supported shell commands. The last element is set to NULL.

  • cd

  • export
  • unset
  • env
  • exit
